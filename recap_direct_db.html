🔎 How Sentinel reconstructed this
The PR removes the PgBouncer proxy layer between the workers and the database, replacing it with direct connections to Postgres. This goes against the decision made in PR #19 which stated that application workers should not open direct database connections and instead should route all access through PgBouncer in transaction-pooling mode. The purpose of this decision was to decouple the app scale from database session capacity to prevent overwhelming the database during autoscaling events, as observed in the incident documented in Issue #11.
